Utilisation de la librairie tekgui



This manual is made for user who know how to use the following function :

bunny_set_key_response();
bunny_set_loop_main_function();
bunny_loop();



Establishment of an ini file



The tekgui library need a ini file to work.
This is necessary to create a good one.
Some part of the ini file are mandatory, some other are customizable.
The ini file must be set like the following example.
If not, some function of the library will return NULL and print an Error.



Example :

[info]
name="window"
height=800
width=800
color=200,200,200
max_ram=15


The domain [info] is a mandatory field.
The domain window contain the name of the window made by the library.
The domain height and width contains the height and the width of the window.
The domain color contain the color of the window.
The domain max_ram contain the maximum of ram usable by the program.

If you see Error: error in malloc print on the terminal, it's probably because max_ram is too short.
If window, color, height or width is empty or miss, the function will return an error.



Initialization of the ini file



This part is used to load the ini informations and set the tekgui structure.

Example :

[window]
nbr_button=3
name_button=button1,button2,button3
nbr_textbox=2
name_textbox=textbox1,textbox2
nbr_checkbox=2
name_checkbox=checkbox1,checkbox2
nbr_picbox=2
name_picbox=picbox1,picbox2


The domain [window] is a mandatory field.
The domain nbr_button is the number of button in the window.
The domain name_button contain the name of all buttons. The number of name and nbr_button must match.
The domain nbr_textbox is the number of textbox in the window.
The domain name_textbox contain the name of all textbox. The number of name and nbr_text must match.
The domain nbr_checkbox is the number of checkbox in the window.
The domain name_checkbox contain the name of all checkbox. The number if name and nbr_checkbox must match.
The domain nbr_picbox is the number of picbox in the window.
The domain name_picbox contain rhe name of all picbox. The number if name and nbr_checkbox must match.

WARNING All the domain must be set.
If, for example, you don't want checkbox in you interface, you should do :

[window]
nbr_button=3
name_button=button1,button2,button3
nbr_textbox=2
name_textbox=textbox1,textbox2
nbr_checkbox=0
name_checkbox=NULL
nbr_picbox=2
name_picbox=picbox1,picbox2



Button setup



Each button must be define as below :

Example :

[button1]
name="salut"
posx=0
posy=0
height=100
width=100
color=70,70,70
select_color=150,150,150
fct_to_call="button1"


The domain [button1] contain the name of the name of the button to set, surrounded by [ ].
The domain name contain the name of button in the window.
The domain posx and posy contains the starting position of the button.
The domain height and wight define the size of the button.
The domain color contain the color of the button.
The domain select_color is the color when the mousse is on the button.
The domain fct_to_call contain the name of the function to call when you clock on the button.

Each button must have is own setup.
If there is two different setup for the same button, the program will use only the first one.



Textbox setup



Each textbox must be define as below :

Example :

[textbox1]
name="tekbox1"
posx=300
posy=100
height=100
width=300
color=80,80,80
text="bonjour"


The domain [textbox1] contain the name of the textbox, surrounded by [ ].
The domain name contain the name of the textbox. The name will not be print on the window.
The domain posx and posy define the starting position of the textbox.
The domain height and wight define the size if the textbox.
The domain color contain the color of the textbox.
The domain text contain the initial string print in the textbox

Each textbox must have is own setup.
If there is two different setup for the same textbox, the program will use only the first one.
To use the textbox, you just have to click on it.
Only the alphabetical character, the space and delete can be use in textbox.
The other character will have no effect.



Checkbox setup



Each checkbox must be define as below :

Example :

[checkbox1]
name="checkbox1"
posx=700
posy=70
height=50
width=50
fct_to_call_check="check"
fct_to_call_not_check="not_check"


The domain [checkbox1] contain the name of the Checkbox, surrounded by [ ].
The domain name contain the name of the checkbox. The name will not be print on the window.
The domain posx and posy define the starting position of the checkbox.
The domain height and wight define the size if the checkbox.
The domain fct_to_call_check contain the name of the function to call when the checkbox is check.
The domain fct_to_call_not_check contain the name of the function to call when the checkbox is uncheck.

Each checkbox must have is own setup.
If there is two different setup for the same checkbox, the program will use only the first one.



picbox setup



picbox mean picture box.
Each picbox but be define as below.


Exemple :

[picbox1]
name="emoji6.png"
posx=100
posy=250


The domain [picbox1] contain the name of the picbox, surrounded by [ ].
The domain name contain the link of the picture.
The demain posx and posy contain the starting position of the pixture.

Each picbox must have is own setup.
If there is two different setup for the same checkbox, the program will use only the first one.
The picture will be print from the starting position to the end.
If the end of the picture is out of the window, an arror will be print.
Name must contain the link to the picture, or the function will retrun an error.
If the pitecute not exist, or the link is false, the function will retrun an error.
