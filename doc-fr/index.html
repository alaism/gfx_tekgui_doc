Utilisation de la librairie tekgui



Ce guide est fait pour les utilisateurs qui connaissent les bases de la Liblapin, et qui maîtrisent les fonctions :

bunny_set_key_response();
bunny_set_loop_main_function();
bunny_loop();



Création d'un fichier ini



La librairie tekgui se base sur un fichier ini pour créer une interface graphique personalisée.
Il est donc nécessaire de créer un fichier ini correctement.
Certaine partie sont obligatoires, d'autre sont personalisables. Le fichier ini doit être configuré comme dans l'exemple ci-dessous.
Dans le cas contraire, les fonctions de la librairie tekgui vont renvoyer une erreur.


Exemple :

[info]
name="window"
height=800
width=800
color=200,200,200
max_ram=15


Le champ [info] est obligatoire.
Le champ window contient le nom de la fenêtre créée par la librairie.
Les champs height et width contiennent la longueur et la largeur de la fenetre.
Le champ color contient la couleur de la fenetre.
Le champ max_ram contient le maximum de ram utilisable par la programme.
Si l'erreur Error: error in malloc apparait, c'est probablement que max_ram est trop petit pour supporter la librairie Tekgui.
Si window, color, height ou width n'est pas complété ou existant, les fonctions retourneront une erreur.



Initialisation du fichier ini



La suite du fichier ini contient les informations principales de la fenêtre.

Exemple :

[window]
nbr_button=3
name_button=button1,button2,button3
nbr_textbox=2
name_textbox=textbox1,textbox2
nbr_checkbox=2
name_checkbox=checkbox1,checkbox2
nbr_picbox=2
name_picbox=picbox1,picbox2


Le champ [window] est obligatoire.
Le champ nbr_button est le nombre de bouton présent dans la fenètre.
Le champ name_button contient les nom des boutons. Le nombre de nom et nbr_button doivent etre les même.
Le champ nbr_textbox est le nombre de case de text présent dans la fenêtre.
Le champ name_textbox contient les noms des cases de text. Le nombre de nom et nbr_textbox doivent etre les même.
Le champ nbr_checkbox est le nombre de checkbox présent dans la fenêtre.
Le champ name_checkbox contient les noms des cases de text. Le nombre de nom et nbr_checkbox doivent etre les même.
Le champ nbr_picbox est le nombre d'image présent dans la fenêtre.
Le champ name_picbox contient les noms des images. Le nombre de nom et nbr_picbox doivent etre les même.
ATTENTION Tous les champ doivent apparaitre dans le fichier ini.
Si, par exemple, vous ne voulez pas de checkbox, voici comment faire :

[window]
nbr_button=3
name_button=button1,button2,button3
nbr_textbox=2
name_textbox=textbox1,textbox2
nbr_checkbox=0
name_checkbox=NULL
nbr_picbox=2
name_picbox=picbox1,picbox2



Configuration des boutons



Apres avoir été initialisé, chaque bouton doit être defini comme dans l'exemple ci-dessous.

Exemple :

[button1]
name="salut"
posx=0
posy=0
height=100
width=100
color=70,70,70
select_color=150,150,150
fct_to_call="button1"


Le champ [button1] contient le nom du bouton que vous voulez definir, entourée de [ ].
Le champ name contient le nom du bouton dans la fenêtre.
Les champ posx et posy contiennent la position de départ du bouton.
Les champ height et wight définissent la taille du bouton.
Le champ color est la couleur du boutton.
Le champ select_color est la couleur lorsque le bouton est selectionné.
Le champ fct_to_call contient le nom de la fonction appeller lorsque le bouton est cliquer.
Chaque bouton doit avoir sa propre configuration.
Si il y a deux configurations différentes pour le même bouton, la première configuration trouvé sera activée, la seconde sera ignorée.



Configuration des boîtes de texte



Apres avoir été initialisé, chaque boîte de texte doit être définie comme dans l'exemple ci-dessous.

Exemple :

[textbox1]
name="tekbox1"
posx=300
posy=100
height=100
width=300
color=80,80,80
text="bonjour"


Le champ [textbox1] contient le nom de la boite que vous voulez definir, entourée de [ ].
Le champ name contient le nom de la textbox dans la fenêtre.
Les champ posx et posy contiennent la position de départ de la textbox.
Les champ height et wight définissent la taille de la boite.
Le champ color contient la couleur de la textbox.
Le champ text contient l'écriture initial à l'intérieur de la boite.

Chaque boite de texte doit avoir sa propre définition.
Si il y a deux configurations différentes pour la même boite, la première configuration trouvé sera activée, la seconde sera ignorée.
Pour utiliser la boite de texte, il suffit de cliquer dessus.
Seul les caractères alphabétiques, les espaces et la touche retour arrière peuvent être utilisé dans ses boîtes
Les autres carractères n'auront uncun effet.



Configuration des Checkbox



Apres avoir été initialisé, chaque checkbox doit être définie comme dans l'exemple ci-dessous.

Exemple :

[checkbox1]
name="checkbox1"
posx=700
posy=70
height=50
width=50
fct_to_call_check="check"
fct_to_call_not_check="not_check"


Le champ [textbox1] contient le nom de la checkbox que vous voulez définir, entourée de [ ].
Le champ name contient le nom de la checkbox dans la fenêtre.
Les champ posx et posy contiennent la position de départ de la checkbox.
Les champ height et wight définissent la taille de la checkbox.
Le champ fct_to_call_check contient le nom de la fonction à appller lorsque la case est cochée
le chemp fct_to_call_not_check contient le nom de la fonction à appeller lorque la case n'est pas cochée

Chaque checkbox doit avoir sa propre définition.
Si il y a deux configurations différentes pour la même checkbox, la première configuration trouvé sera activée, la seconde sera ignorée.
Il suffit de cliquer sur la checkbox pour changer sont etat.



Configuration des picbox



Apres avoir été initialisé, chaque picbox doit être defini comme dans l'exemple ci-dessous.
Le picbox veut dire Picture box, ou "boite d'image"

Exemple :

[picbox1]
name="emoji6.png"
posx=100
posy=250


Le champ [picbox1] contient le nom de la picbox que vous voulez definir, entourer de [ ].
Le champ name contient le lien de l'image a charger.
Les champ posx et posy contiennent la position de départ de la checkbox.

Chaque picbox doit avoir sa propre definition.
Si il y a deux configurations différentes pour la meme picbox, la première configuration trouvé sera activée, la seconde sera ignorée.
L'image sera affiché à partir des coordonnées donné, jusqu'a sa fin.
Si la fin de l'image se trouve après la fin de la fenètre, une erreur apparaitra dans la terminal.
name doit contenir un lien direct ou le chemin d'accés à votre image à partir de la racine de la librairie.
Si l'image donnée n'existe pas ou que le lien est mal entée, la fonction retournera une erreur.
